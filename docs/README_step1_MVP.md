# Nurse Handover Assistant - Step 1: 데이터 구조 정의 (MVP)

## 목적 (Goal)

이 단계의 목표는 **환자 데이터를 일관된 구조로 정의**하는 것이다.\
이 구조는 이후 단계에서:

-   이전 데이터 저장
-   현재 데이터 입력
-   변화 감지 (Change Detection)
-   인수인계 요약 생성

의 기준이 되는 **핵심 기반 (Foundation)** 역할을 한다.

------------------------------------------------------------------------

## Step 1에서 구현할 것

1.  환자 데이터 구조 설계
2.  JSON 형태로 저장 가능하도록 설계
3.  Change Detection이 가능하도록 비교 가능한 구조 유지

이 단계에서는:

-   데이터베이스 사용하지 않음
-   UI 만들지 않음
-   복잡한 클래스 구조 만들지 않음

**단순하고 안정적인 dict 구조**를 사용하는 것이 목표이다.

------------------------------------------------------------------------

## 기본 데이터 구조 (MVP)

``` python
patient_data = {
    "patient_id": "P001",

    "vitals": {
        "bp": "120/80",
        "hr": 72,
        "temp": 36.5
    },

    "medications": [
        "Amlodipine"
    ],

    "notes": [
        "통증 없음"
    ]
}
```

------------------------------------------------------------------------

## 필드 설명

### patient_id

    환자를 식별하는 고유 ID

예:

    P001
    P002
    P003

------------------------------------------------------------------------

### vitals (dict)

Vital sign은 key-value 구조이므로 dict를 사용한다.

예:

    BP
    HR
    TEMP
    RR
    SpO2

예시:

``` python
"vitals": {
    "bp": "120/80",
    "hr": 72,
    "temp": 36.5
}
```

------------------------------------------------------------------------

### medications (list)

Medication은 여러 개 존재할 수 있으므로 list를 사용한다.

예:

``` python
"medications": [
    "Amlodipine",
    "Lasix"
]
```

------------------------------------------------------------------------

### notes (list)

특이 이벤트나 임상 메모를 기록하는 필드이다.

예:

``` python
"notes": [
    "발열 있음",
    "낙상 없음",
    "통증 호소"
]
```

------------------------------------------------------------------------

## 왜 dict를 사용하는가?

이 단계에서는 class보다 dict가 더 적합하다.

이유:

1.  JSON 저장이 매우 쉬움
2.  구조 변경이 쉬움
3.  디버깅이 쉬움
4.  MVP 개발 속도가 빠름

나중 단계에서:

    class Patient
    class VitalSigns

로 리팩토링할 수 있다.
